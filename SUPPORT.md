# Support

## Where to Ask for Help

- Use GitHub issues for reproducible bugs, regressions, and concrete feature requests
- Use [CONTRIBUTING.md](CONTRIBUTING.md) for development workflow expectations
- Use [SECURITY.md](SECURITY.md) for private vulnerability reporting

## Before Opening an Issue

- Confirm you are using the latest tagged release or current `main`
- Check for existing issues covering the same problem
- Include your OS version, app version or commit, and reproduction steps
- Attach logs, screenshots, or screen recordings when they help explain the problem

## What to Expect

Cells is maintained as an early-stage project, so response times may vary. Clear, reproducible reports are much easier to act on than general bug descriptions.
